Hexo 常用命令说明

建立文章

使用 hexo new title 命令来建立文章，这种建立方法会将新文章建立在/source/_posts 目录下,会生成title.md文件
举个栗子：hexo new "my first article"
简写：hexo n "my first article"

生成文章静态文件

使用 hexo generate 命令编译/source/_posts 目录下 markdown 文件时，会将其 HTML 结果编译在 public 目录下
注意 该命令会编译所有/source/_posts下的md文件
简写 hexo g

部署文章

使用 hexo deploy 将会把 public 目录下所有文章部署到 GitHub
注意 该命令会发布public目录下的html文件到github
该命令简写为 hexo d

特别注意：若我们同时编辑多篇文章，只要其中一篇文章尚未编辑完成，也会随着 hexo deploy 一起部署到 GitHub，也就是 GitHub 可能会看到我们尚未完成的文章

如何避免上述的问题呢？我们可以先写文章，写完以后在发布

建立草稿

使用 hexo new draft <title> 新文章将建立在 source/_drafts 目录下，因此 hexo generate 并不会将其编译到 public 目录下，所以 hexo deploy 也不会将其部署到 GitHub
举个栗子：hexo new draft "my first draft"
简写：hexo n draft "my first draft" 会生成my-first-draft.md文件

预览草稿

使用命令 hexo server --draft 前提是我们已经执行过hexo g生成过静态文件了，这时候hexo服务会在我们本机启动，默认端口4000，我们访问localhost:4000即可看到自己的草稿了
简写：hexo s --draft

如何一边写草稿，一边预览呢？我们需要安装插件hexo-browsersync
首先得到hexo工作的根目录下 我的是D://hexo_backup/
安装命令：npm install hexo-browsersync --save
安装完以后执行hexo s命令启动，会看到 UI: http://localhost:3001和UI External: http://localhost:3001
这说明插件已经安装启动好了，访问http://localhost:3001 这是异步浏览配置的控制台页面，可以在这里进行一些自定义的配置(默认不用配置)。
我们访问http://localhost:4000 可以预览我们的文章，这时候去编辑预览文章的md文件，然后保存，会看到页面自动刷新以及我们改变的内容

发布草稿

使用命令 hexo publish <filename> 该命令是将文章从 source/_drafts 移动到 source/_posts，然后我们在执行编译和发布文章命令即可。也可以手动剪切source/_drafts下的文章，粘贴到source/_posts中也是一样的
注意：filename是不带 不带 不带 后缀.md的，只是文件名
举个栗子： hexo publish my-first-draft 就是把my-first-draft.md文件从 source/_drafts 移动到 source/_posts
简写：hexo p my-first-draft

删除文章

先删除本地md文件，然后通过生成和部署命令进而将远程仓库中的文件也一并删除。
假如我不小心把source/_posts下my-first-article.md文章部署到github了，需要删除该文章
操作如下：
1.到source/_posts下删除my-first-article.md文件
2.回到hexo工作的根目录执行命令hexo d -g 即可将已经发布的文章删除

清除缓存

使用 hexo clear 命令清楚缓存，删除public文件夹及其下所有的文件
简写：hexo cl

一次执行多个命令

hexo中可以使用&& 来批量执行多个命令
部署文章：hexo cl && hexo g && hexo d 先清缓存，再编译生成静态文件，再部署到github
预览文章：hexo cl && hexo g && hexo s --draft 先清缓存，在编译生成静态文件，再启动服务预览草稿

其他环境相关的命令

查看npm的镜像源

使用 npm config get registry

切换淘宝的镜像源

使用 npm config set registry https://registry.npm.taobao.org

切回默认的镜像源

使用 npm config set registry https://registry.npmjs.org

查看版本

使用 hexo version 查看hexo的版本
简写：hexo v

查看过期的依赖及版本

使用 npm outdated 查看过期的依赖

查看已安装的插件

使用 npm ls -dept 0 查看已安装的插件和版本号
