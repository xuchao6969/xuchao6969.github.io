hexo 博客备份

搭建完个人博客后，考虑到万一哪天自己的写过的文章丢失了怎么办？第一反应就是弄个仓库，提交到仓库上，仓库总不会倒闭吧，最保险的还是上传到github的私人仓库~~~

1、github创建私人仓库hexo_backup
2、把本地博客的内容全部上传到hexo_backup 主要是scaffolds、source、themes等文件夹下所有内容和_config.landscape.yml、_config.yml、package-lock.json、package.json文件，其他随意
3、每次写完后，提交、推送到github

备份恢复

假如本地博客工作空间被删除了，从git上down下来以后怎么恢复呢？

1、使用git clone命令将备份仓库下载到本地，相当于创建了博客本地文件夹
2、在博客文件夹下执行命令 npm install hexo-cli -g 安装hexo插件
3、执行命令 npm install 自动安装依赖文件*（如butterf主题的依赖、Gulp插件等，若未自动安装成功，手动安装即可）
3、执行命令 npm install hexo-deployer-git --save 安装部署插件

验证：在本地博客文件夹下运行cmd 执行hexo cl && hexo g && hexo s 访问本地路径localhost:4000 看看博客的内容、主题、样式是否和之前一样

进阶

每次写完文章，都要提交、推送，有点麻烦怎么办？经过了解，可以使用hexo-git-backup插件实现博客备份。

1、创建备份仓库，在Github上新建私有仓库(如果自己有，可以使用自己的私人备份仓库hexo_backup)
2、在hexo配置文件(_config.yml)中添加以下内容：注意写自己的github用户名和对应的分支号，需要注意的是以前创建的仓库默认分支是master后来改为main了

3、安装博客备份插件npm install hexo-git-backup --save
4、写完文章后运行备份命令hexo backup或者hexo b 即可自动备份本地博客文件家下的内容到git
